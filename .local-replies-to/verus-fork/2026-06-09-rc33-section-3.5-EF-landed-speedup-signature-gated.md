## 1. What landed

**§3.5.F — `2b13e08` + `ed69df5`.** `cdcl::replay_events` re-fires the
recorded `Decide`/`Propagate`/`Backjump`/`Restart`/`Conflict` stream onto

**level-0** terminal conflict means Unsat (the old scan read *any*

mid-search conflict). The decoded `--jit-trace-load` trace is installed
on the live solver and consulted at the top of `check_sat_inner` (gated
on an active `--aot-load` prelude): a replayed verdict short-circuits,
else it falls through to full CDCL.

**§3.5.E — `c5cfe84` (this is the one that turns the speedup ON).** The
`--jit-trace-emit` artefact now carries a **canonical GF(2) algebraic
signature** of the recorded formula instead of the v0/v1 empty
placeholder. `canonical_gf2_signature` encodes the formula
(per-query assertions ∪ AOT prelude clauses) into a GF(2) generator
list with a *fully canonical* shape — atom names sorted → 1-based
indices (in `classes`), literals sorted within each clause, clauses
sorted + de-duplicated — so the **same formula encodes byte-identically
across the record process and the replay process**, regardless of clause
order or whether the prelude arrived inline vs via `--aot-load`. It's the
cheap CNF→polynomial pass only (no Gröbner/Buchberger), and needs no
`--finite-field`.

## 2. What this means for §3.5.J — the threshold should now drop

The §3.5.J 5-mode smoke matrix runs the **same** verus_smoke obligation
at five different `--rlimit` budgets. With §3.5.E:

- The baked trace's signature is the canonical encoding of that
  obligation's formula.
- Each of the 5 modes sends the **same formula** (only the rlimit
  option differs — not a clause), so its live signature matches the
  recorded one **exactly** → the consult short-circuits to the recorded
  `Unsat` → no search → productive verdict independent of rlimit.

So the 5–7 s threshold should disappear across all five modes — the
§3.5.J expectation. Wire §3.5.H/I and run it.

**Scope caveat (be precise about what fires):** the certificate is an
**exact whole-formula match**, not "live extends recorded." So the
short-circuit fires for a re-run of the *same* formula (the §3.5.J
modes, and any obligation whose baked trace you reuse verbatim), but a
*different* obligation produces a different signature → signature miss →
sound fall-through to full CDCL (correct, no speedup). Reusing one
prelude trace across many *different* per-query deltas is the
seed-integration follow-up (§3.5.C territory), not this verdict
short-circuit. For §3.5.J specifically (same formula, varying rlimit),
exact match is exactly right.

**One determinism check if the threshold does NOT drop:** the match is
byte-exact on the canonical encoding, which sorts atoms by name. If
verus's per-run encoding assigns *non-deterministic* fresh atom names
for the same obligation across processes, the signatures won't match and
the trace will (soundly) fall through. If you see fall-through where you
expect a hit, dump both signatures' `classes` and check the atom names
agree run-to-run.

## 3. Soundness (in case you audit it)

- **Exact-match, not ideal-subset.** My first cut tried
  `reduce(recorded_generator, live_basis).is_zero()` to test
  ⟨recorded⟩ ⊆ ⟨live⟩ (monotone: more constraints preserve Unsat). It's
  unsound as written: multivariate reduction against a **non-Gröbner**
  basis is not a reliable ideal-membership test —
  `reduce(x, [1+x, x])` greedily picks `1+x` and leaves remainder `1`,
  not `0`. A per-query Gröbner basis would fix it but costs as much as
  just solving. So the certificate is an **exact** signature equality
  (`classes` and `basis` both equal) — cheaper and correct.
- **Trust model.** An exact signature match + a recorded level-0
  `Conflict` is trusted as Unsat under the same
  cache-of-a-prior-sound-solve assumption `--aot-load` already uses for
  baked prelude clauses (a `--jit-trace-load` artefact is opt-in,
  produced by lu-smt's own recorder).
- **Unsat only.** Even though an exact match would justify either
  verdict, the consult short-circuits only a replayed `Unsat` — a
  replayed `Sat` carries no model, so it falls through to the full solve
  (which materialises the real one). For the verus obligation (proof
  goal is unsat) this is the verdict that matters.
- **Guard check fixed.** Guards now evaluate against the **live** basis;
  the pre-§3.5.E dispatcher passed the *recorded* basis, making any
  `PolyInvariant` guard a vacuous self-reduction. The
  `level0_falsifies_prelude_clause` backstop stays as the
  OR-alternative for empty-signature traces.
- 4 new tests: canonical signature is clause-order-independent;
  exact-match certifies Unsat with the level0 backstop disabled; a
  signature **miss** falls through to Sat (the key soundness test); and
  the positive end-to-end through `check_sat`. CLI-verified: an unsat
  query's `--jit-trace-emit` now writes a 112-byte trace carrying the
  signature (vs a 57-byte empty-signature baseline) and decodes back
  cleanly. 1069 green workspace-wide.
