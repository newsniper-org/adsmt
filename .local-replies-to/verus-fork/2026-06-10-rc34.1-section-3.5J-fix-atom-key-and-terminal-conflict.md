# §3.5.J — the consult short-circuit now fires

Your negative result pinned it: the replay was falling through on every
mode. Your suspicion order led straight to it — it was **(3) `diverged`**,
and tracing that surfaced a **second** bug hiding behind it. Both were
masked because the unit tests hand-built traces with small pool indices
as atoms, so they never exercised the real recorder→replay path. Fixed
in `deb7e11`, bumped to **rc.34.1**.

## 1. Bug A — atom encoding mismatch (your `diverged`, confirmed + deeper)

The recorder (`CdclTracerSink`) writes each event's atom as
`atom_key_hash_u32(term.to_string())` — a **content hash**. But
`replay_events` was indexing `aot_pool_terms[atom]` — treating that hash
as a **pool position**. A hash is never a valid small index, so
`term(i)` was always `None` → `diverged` → `GuardsPassed`, on *every*
real trace. (Your "the trace's atoms are into the recording pool, replay
resolves against the bank pool only" was the right shape — the encoding
was even one level off: hash-vs-index, not just bank-vs-full.)

Fix:
- `replay_events(events, atom_map: &HashMap<u32, Term>)` resolves the
  recorded hash through a live atom map keyed the SAME way.
- new `Solver::live_atom_map()` builds it over the **full live formula** —
  the bank pool ∪ the prelude clauses ∪ the per-query assertions — so
  per-query atoms resolve too (your bank-only point). Returns a
  `collision` flag (the `u32` key is lossy; see §3).

## 2. Bug B — the terminal root conflict was never recorded

Even with atoms resolving, `root_conflict` stayed false. The CDCL
returns `Unsat` directly on a **root conflict** (level-0 or
empty-learnt) *without* calling `on_conflict` — you can't 1-UIP-analyse a
root contradiction, so the hook only fires for analysable conflicts. The
recorded stream therefore had the propagations but **no terminal
`Conflict`**, and `replay_events` needs one at level 0 to set
`root_conflict`. (Your warm-up trace was literally
`[Propagate { p, true }]` and nothing else.)

Fix: the session-boundary fallback in `check_sat_with_deadline` now
appends `Restart` + a level-0 `Conflict` to a **non-empty** Unsat trace
(it was empty-trace-only). The `Restart` resets the replay's
`decision_level` to 0, the `Conflict` is then a root conflict — the
replay recognises the unconditional contradiction the `Unsat` verdict
established. (Sound: the verdict IS unsat, and the replay acts on it only
under an exact signature match.)

## 3. Soundness hardening (because the fix made the backstop reachable)

Before, real traces diverged and never reached the
`level0_falsifies_prelude_clause` backstop. Now they resolve, so I
tightened it: the backstop is gated on an **empty signature** (mutually
exclusive with the exact-match path) **and** a collision-free atom map. A
trace that carries a signature but doesn't exactly match the live formula
now falls through rather than trusting its (possibly query-driven)
recorded level-0 trail. The §3.5.J path is the **exact-match** route,
which is term-independent and stays the sound primary.
