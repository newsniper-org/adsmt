## 1. The `--emit-cert` producer surface ŌĆö your `-V emit-*` hook target

The Y4 R7.11 chain (Verus ŌåÆ adsmt-cert ŌåÆ Isabelle/Rocq) needs a way to
get the proof certificate **out** of a `-V adsmt` run so the emitters
can consume it. rc.32 (`7faa073`) lands exactly the producer side your
`-V emit-isabelle` / `-V emit-rocq` dispatch forwards into:

```
--emit-cert <PATH>          write the cert of each unsat (check-sat) (last wins)
--emit-cert-dir <DIR>       one file per unsat ŌåÆ <DIR>/<seq>.cert.<ext>
                            ŌåÉ THIS is the ADSMT_CERT_DIR hook target
--emit-cert-format <cbor|json>   cbor default (the emitters' wire); json for inspection
```

So the `ADSMT_CERT_DIR` env hook in your emit dispatch (per Y4's
`emit-cert-hooks-plus-jit-flag-wire` pair request ┬¦3.2) maps to
`--emit-cert-dir <ADSMT_CERT_DIR>` on the lu-smt invocation. The cert
is the canonical `adsmt-cert::Certificate` (CBOR by default), written
only on `unsat` (the proof-carrying case). `Theorem` is deliberately
non-deserializable, so a cert can't reconstruct a kernel trust token ŌĆö
the emitters re-derive from the stored `Sequent`s.

That's the half on adsmt's side; the `-V emit-*` flag + the
argv/env forwarding is yours to land.
