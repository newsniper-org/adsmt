# The bytes were in the signature — fixed (rc.34.3)

Your measurement was exactly right and the slim lever I shipped was one
layer too shallow: on a prelude trace the dropped event stream is 0.6%,
the §3.5.E GF(2) signature is the 99.4%. Thanks for measuring it instead
of assuming. rc.34.3 moves the certificate to where the cost is.

## The fix — a 32-byte canonical clause-set digest

The exact-match certificate is no longer the megabyte GF(2) `basis`. It's
a **32-byte KangarooTwelve-256 digest of the canonical clause set**
(`Solver::jit_trace_digest`, via `lu_common::k12` — a new `adsmt-engine`
dep). This hits **both** of your angles at once, because the clause-set
hash *is* the digest:

- **size / load / compare** — the `basis` is dropped from the wire (full
  AND slim traces now carry an empty `GF2Snapshot` + the digest). The
  `.lutrace` bumps to **v2** with a trailing `signature_digest:
  Option<[u8;32]>`; `read_trace` still loads v1 (digest `None`). The
  3.5 MB load and the megabyte byte-compare both collapse to 32 bytes.
- **compute** — exactly your clause-set-hash insight: for exact-match you
  don't need the GF(2) polynomial structure. `jit_trace_digest` hashes
  the canonical clause set (`canonical_clause_set`: sorted atom names +
  sorted/de-duplicated DIMACS — the same canonical form, factored out of
  `canonical_gf2_signature`) and **skips `cnf_to_generators` entirely**.
  The consult now computes only the cheap digest;
  `canonical_gf2_signature` is computed lazily — only when a trace
  carries guards, which §3.5.E/J traces never do.

So the consult cost is now: **(load 32 B) + (canonical-clause-set hash of
the live formula) + (32-byte compare)** — no megabyte load, no megabyte
compare, no polynomial encoding. The break-even drops from "search >
~0.45 s" to "search > the clause-set pass."

## Soundness — unchanged trust model

Same exact-formula-match trust as before, via a collision-resistant hash:
two formulas with the same canonical clause set hash equal; distinct
formulas hash distinct with overwhelming probability (K12-256). The
consult exact-match is `trace.signature_digest == Some(self.jit_trace_digest())`;
a legacy v1 trace (no digest) falls back to the GF(2) `(classes, basis)`
equality; the `level0_falsifies_prelude_clause` backstop is gated on
*no* exact-match certificate (no digest AND no basis), so a
digest-carrying trace never relies on it. Three new regressions: the
digest is clause-order-independent + formula-sensitive, a digest-only
trace short-circuits Unsat without any GF(2) basis, and the v2 wire
round-trips the digest.
