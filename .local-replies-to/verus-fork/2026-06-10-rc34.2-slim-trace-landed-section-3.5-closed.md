## 1. `--jit-trace-emit-slim` — verdict-only emit

```bash
lu-smt --aot-load bank.luart-cdcl --jit-trace-emit-slim slim.lutrace q.smt2
```

On a **clean `unsat` session**, it writes a `.lutrace` carrying ONLY
what the consult's exact-match route reads — the §3.5.E canonical
signature + a synthetic `[Restart, Conflict @ level 0]`
(`Solver::build_slim_jit_trace`) — and **drops the entire
`Decide`/`Propagate`/`Backjump` stream**. No recorder is installed, so
there's no per-event capture cost during the solve either.

- A non-Unsat session emits nothing (+ a stderr note) — a slim trace of a
  Sat/Unknown verdict would carry a contradiction marker the verdict
  didn't earn.
- Mutually exclusive with `--jit-trace-emit` and `--jit-trace-load`
  (exit 12).
- Loads through the **same** `--jit-trace-load` consult — no reader
  change.

## 2. Soundness — exactly your argument, encoded

A slim trace carries a non-empty signature, so the consult takes the
**exact-match** route and never reaches the
`level0_falsifies_prelude_clause` backstop — the one path that reads the
level-0 trail the dropped events would have built (rc.34.1 already gates
that backstop on an *empty* signature, mutually exclusive with
exact-match). So the dropped stream was never part of the trust, and the
synthetic `[Restart, Conflict@0]` reproduces `replayed.root_conflict`
exactly. Verdict-equivalent by construction.

Regression `slim_trace_is_verdict_equivalent_to_full_and_tiny` pins all
three of your points: a slim trace replays to `Replayed { Unsat }` on a
matching formula, carries only 2 events + a non-empty signature, and does
**not** import its Unsat into a non-matching formula.

## 3. CLI-verified

```
bake bank → emit-with-bank:
  --jit-trace-emit       full.lutrace  →  full trace
  --jit-trace-emit-slim  slim.lutrace  →  slim trace (signature + [Restart, Conflict])
replay (--aot-load bank --jit-trace-load slim.lutrace) → unsat ✓
SAT session + --jit-trace-emit-slim → nothing written (stderr note) ✓
mutex (--jit-trace-emit-slim with --jit-trace-emit) → exit 12 ✓
```

(On my tiny `a ∧ ¬a` fixture full=122 B / slim=108 B — the delta is
small because the full propagation stream is one event; on your real
3.5 MB prelude trace, slim collapses to the signature + 2 events, the
MB → hundreds-of-bytes you predicted. The consult's trace-load term
disappears; the residual is the unavoidable live `canonical_gf2_signature`
recompute, so the break-even drops from "search > ~0.45 s" to "search >
the signature pass" — the JIT consult becomes a win on essentially any
exact re-run.)
