# rc.32.1 ‚Äî emitter package manager is now system-installable

A small follow-up to the 2026-06-08 `--emit-cert` reply. **rc.32.1**
is on `testing` and tagged **`v1.0.0-rc.32.1`**. It's a hotfix
revision (no source/cert change ‚Äî the `--emit-cert` flag and the cert
wire are byte-identical to rc.32), and it closes a packaging gap that
matters for your Ask #2.

## What changed

The rc.31 WASM emitter stack added two workspace binaries that **no
PKGBUILD was installing**, so they never reached `/usr/bin`:

- **`adsmt-emit`** ‚Äî the emitter package-manager CLI
  (`install` / `run` / `list` / `pack`)
- **`adsmt-env`** ‚Äî the build trampoline (`$srcdir`/`$pkgdir`)

rc.32.1 folds both into the existing **`adsmt-cli[-hpc|-multi][-testing|-git]`**
split package (all 9 PKGBUILDs across the 3 families √ó 3 channels). So
on a packaged install:

```
pacman -S adsmt-cli-testing        # ‚Üí /usr/bin/{lu-smt, adsmt-emit, adsmt-env}
#   (or adsmt-meta-testing for the full logicutils + cli + lsp + ffi set)
```

## What this unlocks for R7.11

Your Ask #2 wanted the emitters system-installed. There are now two
clean paths, and rc.32.1 makes the second one turnkey:

- **(a) native bins** ‚Äî `cargo install --git ‚Ä¶adsmt-contrib --branch
  testing adsmt-emit-rocq-wasm adsmt-emit-isabelle-wasm` ‚Üí `rocq` /
  `isabelle` (stdin CBOR ‚Üí stdout proof). Unchanged from the last
  reply; least friction for a `just emit-rocq` shell-out.
- **(b) WASM package route** ‚Äî now that `adsmt-emit` is on `PATH` from
  `adsmt-cli-testing`, no separate `cargo install` of the PM:

  ```
  adsmt-emit install ~/adsmt-contrib/adsmt-emit-rocq-wasm/rocq.adsmt-emit
  lu-smt --emit-cert cert.cbor proof.smt2
  adsmt-emit run rocq --cert cert.cbor > out.v     # wasmi, sandboxed, -j N
  ```

The contrib emitters themselves (`rocq.adsmt-emit` / `isabelle.adsmt-emit`,
commit `cbf7a46`) are unchanged and stay wire-compatible.

## Pin

- adsmt ‚Üí **`v1.0.0-rc.32.1`** (tag) or `fc7351b` ‚Äî on `testing`.
- adsmt-contrib ‚Üí `cbf7a46` (unchanged).

Update `unified-toolkit-pin.lock` to the rc.32.1 tag and you're current.
Nothing in your verify/emit flow needs to change beyond the pin ‚Äî the
cert wire and `--emit-cert*` surface are the same as rc.32.
