## 1. 세션 재개: oxiz fork의 strict-superset 위반 정정

이전 세션 끝에서 사용자가 강한 피드백을 주었음:

> upsteam의 API surface에서 단 하나라도 절대로 빼면 안 된다고 내가 분명히 지시했을텐데?

직전 작업에서 `oxiz-sat::proof::DratProof` / `LratProof`를 generic + default
(`W: Write + Send = BufWriter<File>`)로 리팩터하면서 `#[derive(Debug)]`을 manual
impl로 바꿔 `"<sink>"`를 출력하게 만든 부분이 byte-identity 위반이었음. Audit 결과:

- `new()`, `default()`, `enable(path)`, `Drop`, 메서드 시그너처 → 모두 보존 ✓
- `Debug` 출력 포맷 → **변경됨 (위반)**

수정: manual `impl<W> Debug for DratProof<W>` 제거 → `#[derive(Debug)]`로 복원.
`W: Debug` bound가 자동 추론되며, default-typed form (`BufWriter<File>`)은
`BufWriter<File>: Debug`를 만족하므로 upstream과 byte-identical Debug 출력.

`test_drat_debug_format_default_typed_matches_derive` 추가 — drift 방지 guard.

검증: `cargo test -p oxiz-sat --lib` → **592/592 통과** (589 upstream + 3 new).

PR description (`docs/upstream/oxiz-enable-writer-pr.md`)의 "Compatibility notes"
섹션이 오해 소지 있게 작성되어 있었음 (Default/Debug/new() 가용성이 좁아진 것처럼
서술). 사실은 default-W 케이스 전부 보존됨 → "Strict-superset audit" 섹션으로 교체.

메모리 (`oxiz_relationship.md`)에 "Lesson learned (2026-05-14)" 추가:
strict-superset 검증 시 method signature뿐 아니라 모든 trait impl의 관찰 가능한
출력까지 audit해야 함.

- `emit_lfsc_via_oxiz(clauses)` → 등장 변수를 `(declare vᵢ bool)`로 선언, 그리고
  `(check false)` 절을 추가

`solver.rs`의 unsat path에서 `dimacs_bytes`와 함께 채워짐.

`oxiz-proof` feature 이름 충돌 (의존 패키지명과 동일) 이슈 발견 → adsmt-engine
워크스페이스 루트에서 `cargo test -p adsmt-engine --features oxiz-proof`로 호출하면
정상 동작. 별도 디렉터리에서는 `cannot specify features for packages outside of
workspace` 오류 나옴 (Cargo 동작상 정상).

또 LfscTerm 변종 이름은 `Symbol(...)`가 아니라 `True`/`False` 등 직접 variant로
존재. `check`는 `LfscTerm::False` 사용.

### 3.2 drat-trim cross-check (초기 — binary 미설치 상태)

새 모듈 `adsmt-engine::drat_trim`, feature `drat-trim` (off by default):

- 입력: `clauses: &[Vec<i32>]` + `drat_bytes: &[u8]`
- CNF + DRAT를 임시파일로 작성 → `Command::new("drat-trim").arg(cnf).arg(drat)`
- 결과: `DratTrimResult::{Verified, Rejected{stderr}, BinaryUnavailable, IoError(String)}`
- `drat-trim` 미발견 시 graceful fallback

테스트는 BinaryUnavailable 경로만 검증 가능한 상태로 시작.

### 5.1 drat-trim 실제 검증

수동 sanity check:

```
$ printf "p cnf 1 2\n1 0\n-1 0\n" > /tmp/test.cnf
$ printf "0\n" > /tmp/test.drat
$ drat-trim /tmp/test.cnf /tmp/test.drat
c parsing input formula with 1 variables and 2 clauses
c finished parsing
c found complementary unit clauses: -1
s VERIFIED
```

→ 우리 empty-clause DRAT가 진짜로 drat-trim의 인증을 받음.

테스트 강화: `polarity_contradiction_verifies_when_binary_present`,
`three_clause_unsat_verifies_when_binary_present`,
`satisfiable_input_rejects_empty_clause_proof`,
`solver_unsat_witness_cross_checks_via_drat_trim` (end-to-end).

각 테스트는 `DratTrimResult::BinaryUnavailable`도 허용 (CI에서 drat-trim 없을 때
skip되도록 — 우리 PKGBUILD가 CI에 깔린다는 보장 없음).

### 5.2 Lean reflection (option b)

새 모듈 `adsmt-cert/src/lean_emit.rs`. cert→Lean4 source emit:

- `emit_lean(cert) -> String`
- 각 free Term variable → `axiom <name> : <type>`
- `StepBody::Assume(t)` → `axiom sᵢ : <t>`
- `StepBody::Refl(t)` → `theorem sᵢ : t = t := rfl`
- `StepBody::Trans/EqMp/...` → `theorem ... := <proof-term>` (가능한 케이스),
  나머지는 `:= by sorry`로 두되 statement type은 정확하게 매핑
- `StepBody::Assumed { formula, explain }` →
  `theorem sᵢ : <formula> := sorry  -- abductive: <explain>`
- `StepBody::Theory { ... }` → `axiom sᵢ : <concl>` + witness 요약 주석
- 최종: `theorem result : <conclusion> := s<final>`

Bool 케이스 한정. 추후 cycle에서 HOL+HKT 매핑 확장.

논점: cert side의 `r::assume(b, phi)`는 `KernelResult<ProofHandle>` 반환이라
테스트에서 `.unwrap()` 필요. `ProofHandle.id()` 가 아니라 `.step()` (메서드명
정정 후 통과). `r::refl(b, &t)`는 `&Term` 받음.

### 5.3 Coq export wiring

`oxiz-proof::CoqExporter`는 `oxiz_proof::Proof` (graph type) 입력 받음 — bare
`DratProof`가 아님. 변환 헬퍼 추가:

- `emit_coq_via_oxiz(clauses, drat)`:
  - 각 입력 절 → `Proof::add_axiom(string)`
  - 각 DRAT Add → `Proof::add_inference("rup", premises, conclusion)`
  - Delete는 oxiz-proof Proof에 직접 표현 없음 — no-op로 매핑
  - 최종 `CoqExporter::new().export_proof(&p)`

`TheoryWitness::Drat`에 `coq_bytes` 필드 추가. emit.rs에서 `:coq` 키워드로 출력.
