# The fold is O(delta) now — but it wasn't the only O(formula) term

rc.34.4 landed cleanly: the bank gained the v1.3 fold (224 962 →
225 003 B, +40 B = `([u8;32], u64)`), `.lutrace` unchanged at 99 B, the
bake script needed no change, verdict-independence still holds
(rlimit=1 → `unsat` via consult). Pinned **rc.34.4**, verus rebuilt
(vstd 1690).

But you asked for the numbers if a fixed prelude-scale term remained —
and it does, unchanged:

```
(1) baseline:                      ~0.015 s
(2) --aot-load only:               ~0.019 s
(3) --aot-load + --jit-trace-load: ~0.40  s
consult marginal = (3) − (2)  ≈  0.38 s     ← identical to rc.34.3
```

And the differential confirms it's still O(formula), not O(query delta):

```
consult on a tiny p∧¬p (tiny prelude):     ~0.001 s
consult on the verus prelude warm-up:      ~0.40  s
```

So the incremental fold is correct but it wasn't the (whole) cost.

## Where it actually is — `live_atom_map()`, not the digest

I read `replay_aot_cdcl_trace` (rc.34.4). The digest path is fixed:

- `solver.rs:814` — `canonical_gf2_signature()` is now properly
  guard-gated (`if !trace.guards.is_empty()`), and a §3.5.J trace carries
  no guards, so the polynomial encode is correctly **skipped**. Good.
- The digest fold itself is your new O(delta) path. Also good.

But one line down the replay path is unconditional and O(whole formula):

```rust
// solver.rs:872
let (atom_map, atom_map_collision) = self.live_atom_map();
let replayed = crate::cdcl::replay_events(&trace.events, &atom_map);
```

`live_atom_map()` builds a map over the **bank ∪ per-query** formula on
**every** consult, to resolve the recorded events' content-hash atoms
(the rc.34.1 `diverged` fix). It's the same whole-formula sweep the
digest used to be — just moved one concern over. That's the 0.38 s: it
scales with the prelude (tiny prelude → 0.001 s), and the digest fix
can't touch it because it's the *atom map* for `replay_events`, not the
signature.

## Two levers (your call)

1. **Skip it for a slim trace.** A slim trace's events are exactly
   `[Restart, Conflict@0]` — the synthetic level-0 root conflict. If that
   conflict marker doesn't reference any query atom (it's a bare
   decision-free contradiction), `replay_events` doesn't need the live
   atom map at all to set `root_conflict`. Gate the `live_atom_map()`
   build on "the trace has events that reference atoms" — a slim trace
   then pays nothing here, and the consult drops to ~O(delta) end-to-end.
   (Full traces, whose `Propagate`/`Decide` events do reference atoms,
   keep building it.)
2. **Precompute / cache the prelude's atom-map share** — the exact
   treatment you just gave the digest fold. The prelude atoms are fixed
   across a session; build the bank-side atom map once at `--aot-load`
   (or bake the name→term table into the v1.3 section) and per-consult
   merge only the per-query atoms. `live_atom_map()` then goes O(delta)
   too.

Lever 1 is the cheaper fix and exactly fits §3.5.J (slim traces are the
§3.5.J artefact); lever 2 also helps full traces and the §3.5.C
cross-query case. The differential says either one should finally drop
`(3) − (2)` to the query-delta scale.
