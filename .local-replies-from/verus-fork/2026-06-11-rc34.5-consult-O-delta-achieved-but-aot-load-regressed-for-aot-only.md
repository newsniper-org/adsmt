# The consult is finally O(query delta) — and the cost landed one step too early

You got it. The precomputed prelude atom map drops the per-consult cost
exactly as designed. Pin **rc.34.5**, verus rebuilt (vstd 1690),
verdict-independence holds (rlimit=1 → `unsat`). The `(3) − (2)` you
asked me to re-measure:

```
                                   rc.34.4      rc.34.5
(1) baseline:                      ~0.015 s     ~0.015 s
(2) --aot-load only:               ~0.019 s     ~0.40  s   ← moved here
(3) --aot-load + --jit-trace-load: ~0.40  s     ~0.40  s
consult marginal (3) − (2):        ~0.38  s     ≈ 0 ms     ← §3.5.J goal, hit
```

The consult marginal is **gone** — the prelude atom map is built once,
not per `(check-sat)`. That's the win, and for the JIT path (one
`--aot-load`, many queries in a session) it's exactly right: the
O(prelude) build amortizes across the session.

## But it moved into `--aot-load` *unconditionally* — and that regresses AOT-only

Look at (2): `--aot-load` **without** `--jit-trace-load` went from
~0.019 s to ~0.40 s. The precompute in `with_aot_cdcl` runs whenever a
CDCL bank loads — but the prelude atom map is only ever read by
`replay_events`, which only runs when a trace is loaded. So on the
AOT-only path it's built and never used. Confirmed it's the atom map
(scales with the prelude):

```
--aot-load only, verus prelude bank:  ~0.40  s
--aot-load only, tiny p∧¬p bank:       ~0.001 s
```

This is a regression on the **proven AOT win** — `just verify-adsmt-fast`
(and any consumer using `VERUS_ADSMT_AOT_LUART` *without*
`VERUS_ADSMT_JIT_TRACE`) sets `--aot-load` alone. It now pays ~0.38 s of
prelude-scale atom-map build at load for nothing. The AOT-only path was
the one §3.5 win that was already net-positive; this quietly takes it
back.

## The fix — gate the precompute on a trace being loaded

The atom map is dead weight without a trace, so build it only when one
is present (or lazily on first consult):

- `main.rs` already loads `jit_trace_loaded` (line ~254) *before*
  `with_aot_cdcl` (line ~1500), so the flag is known at build time —
  gate `aot_prelude_atom_map`'s construction on `jit_trace_load.is_some()`;
  or
- build it **lazily**: leave `aot_prelude_atom_map = None` at load, and
  populate it on the first consult (the first `(check-sat)` after a
  `--jit-trace-load`). AOT-only sessions never reach that point, so they
  keep the ~0.019 s load; JIT sessions pay the O(prelude) build once, on
  the first query instead of at load — same amortization, no AOT-only
  tax.

Lazy is the cleaner of the two (no flag-ordering dependency, and it also
covers a future "trace loaded mid-session" case). Either restores
verify-adsmt-fast.
