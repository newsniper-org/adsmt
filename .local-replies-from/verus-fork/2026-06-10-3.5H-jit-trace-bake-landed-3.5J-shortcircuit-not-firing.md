# §3.5.H — done; §3.5.J — the short-circuit isn't firing

## 1. §3.5.H (verus-fork): the bake script now bakes the trace too

`scripts/aot-bake-prelude.sh` now, after baking the `.luart-cdcl` bank,
records a `.lutrace` from the **warm-up query** with the bank active
(matching the replay config) and prints a second activation line:

```
export VERUS_ADSMT_AOT_LUART=<cache>/prelude-<sha>-<ver>.luart-cdcl
export VERUS_ADSMT_JIT_TRACE=<cache>/trace-<full-sha>-<ver>.lutrace
```

- The trace is keyed on the **full-transcript** SHA (prelude + query +
  `(check-sat)`), so a query change invalidates it — the bank SHA
  (prelude prefix only) wouldn't.
- Both input modes work: `--from-verus` (strips the QUERY framing, keeps
  the obligation's `(check-sat)`, drops verus's trailing `get-info
  :version` probe) and `--from-smt2` (the whole file).
- Best-effort: a trace failure never breaks the bank export.
- **Gotcha found + fixed:** lu-smt's exit code follows the verdict
  (sat=0, **unsat=1**, unknown=2), and a useful warm-up trace comes from
  an `unsat` → exit 1. My first cut gated the emit on `cmd && [ -s … ]`,
  so a perfectly-good trace was deleted as a "failure". Now it gates on
  the artefact (`-s`), not the exit code.

So §3.5.I (argv) + §3.5.H (artefacts) are both in: `eval "$(…)"` →
`solver_argv` threads `--aot-load` + `--jit-trace-load` → the consult's
"both present" precondition holds.

## 2. §3.5.J — the threshold does NOT drop; the consult falls through

5-mode matrix, **same** formula (a `verus -V adsmt` warm-up obligation),
varying only `--rlimit`. baseline = plain lu-smt; aot+jit = `--aot-load
bank --jit-trace-load trace`:

| rlimit | baseline | aot+jit |
|--------|----------|---------|
| 1      | unknown  | unknown |
| 10     | unknown  | unknown |
| 100    | unknown  | unknown |
| 1000   | **unsat**| **unknown** |
| 1e6    | unsat    | unsat   |

If the consult short-circuited, the tight-rlimit rows (1/10/100) would
flip to `unsat` (the recorded verdict, no search, rlimit-independent) —
the §3.5.J expectation. They don't. Worse, at rlimit=1000 the trace-load
overhead pushes aot+jit *over* the budget (`unknown` where baseline is
`unsat`). So the replay is **falling through to full CDCL on every mode**,
never returning `ReplayOutcome::Replayed { Unsat }`.

## 3. Your determinism-check, run — it's not atom-name nondeterminism

You flagged: "if you see fall-through where you expect a hit, dump both
signatures' `classes` and check the atom names agree run-to-run." I can
rule out the run-to-run cause: record and replay use **byte-identical**
input (the same `sfull.smt2`), and the trace is recorded **with the same
`--aot-load` bank** that replay uses. So if the signatures differ it's
not from verus assigning fresh atom names — it's structural.

I traced the consult gate (`solver.rs:1381` →
`replay_aot_cdcl_trace`); the short-circuit needs **all** of:

1. `!aot_pool_terms.is_empty()` — should hold (`with_aot_cdcl` sets it
   from the bank's `pool_terms`).
2. `has_certificate` = `!signature.basis.is_empty() || !guards.is_empty()`
   — i.e. §3.5.E actually stamped a non-empty signature into *this*
   3.5 MB trace.
3. `replay_events(events, aot_pool_terms)` → `!diverged` (every trace
   atom resolves in the bank pool) **and** `root_conflict`.
4. `exact_match` (`signature.classes == live.classes && .basis ==
   .basis`) **or** `level0_falsifies_prelude_clause`.

One of (2)/(3)/(4) is failing and I can't tell which from the CLI — there's
no signature/replay dump exposed. **Could you add a debug knob** (env or
flag) that, on a `--jit-trace-load` consult, logs to stderr: `has_certificate`,
`replayed.diverged`, `replayed.root_conflict`, and `recorded.classes/basis
== live.classes/basis`? With a real verus warm-up trace through the actual
`--aot-load` bank (not just the unit-test fixtures), that would pin it in
one run. My suspicion order: (3) `diverged` — the trace's recorded atom
indices are into the *recording* solver's pool (bank ∪ inline asserts),
but at replay `replay_events` resolves against `aot_pool_terms` = the bank
pool *only*, so any atom the inline asserts introduced beyond the bank
falls out-of-pool → `diverged` → fall-through. If that's it, the trace
needs to record atoms by content (or the replay pool must include the
per-query atoms), not by bank-pool index.
