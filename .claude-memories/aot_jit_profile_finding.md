2026-06-13 Phase-1 profile of the "mature adsmt AOT/JIT" initiative (user plan: profile → extract `portable-algebraic-aotjit` → BacCaml-motivated hybrid algebraic JIT).

**Method:** rc.37 native (debug-syms release), prelude-FULL fixture (`.local-replies-from/verus-fork/repro-2026-06-12-prelude-still-unsat/prelude-FULL-still-unsat.smt2`, 1506 lines / 138 assert / 113 forall), `perf record` 3500 samples + cold/warm A/B (30× each). Native returns `unknown` in ~18ms/query (bails fast, no deep search).

**Cold-solve self-time (per-query ~16ms, minus ~1.9ms process startup):**
- type/kind construct + hashcons (`Type::const_`/`app`/`kind_of`/`dest_fun`/`Kind::arrow`) — **27.8%**
- alloc/free/drop (malloc/free + `drop_in_place<Type>`/`<TermInner>` + `Arc::drop_slow`) — **19.7%**
- hashing (`sip::Hasher::write`, `hash_one` — hash-cons lookups) — **15.0%**
- term construct/convert (`term::intern`, `Term::const_`, `convert_expr`) — **12.8%**
- parse driver (`try_fold`/other) — ~11%
- hashtable ops (scc pool buckets) — 4.1%
- **CDCL/engine solve — 2.4%**, **theory EUF/arith — 2.3%**, **JIT replay/consult — ~0%**

**Conclusion: ~75% is term/type-DAG construction + hash-consing; <5% is actual SAT/theory solving.** The §3.5 verdict-replay never even fires here (unknown, no unsat short-circuit).

**Cold vs AOT-load A/B:** COLD ~16.2ms → WARM (`--aot-load` baked prelude + tiny `(check-sat)`) ~6.2ms. **Existing `--aot-load` already removes ~62% of prelude work** by reusing baked state; residual ~6ms = deserialize + re-intern the baked DAG into the hash-cons pool.

**What it means (confirms the user's two intuitions quantitatively):**
1. "진짜 meta-tracing 불필요" ✓ — the noop compiled-kernel (`adsmt-jit/kernel.rs::emit_noop_kernel`, the only emitter; engine `replay_aot_cdcl_trace:882` discards `kernel.invoke()`) and §3.5 verdict-replay both target the <5% solving slice. Native codegen is misdirected.
2. The real lever = reuse the prelude's compiled/interned state across queries — exactly BacCaml's "method-compile the stable region" mapped to SMT: **prelude = stable "method" (bake/intern once, reuse as a unit), per-query goal = "trace" (incremental delta on top).**

**Sharpened plan:** Phase 2 `portable-algebraic-aotjit` extracts the AOT state-reuse + algebraic digest core (NOT the JIT kernel); the "JIT" stays interpreter-replay + digest short-circuit. Phase 3 hybrid = let the per-query trace inline the prelude method's already-interned nodes by handle instead of re-interning (attacks the residual ~6ms re-intern; = BacCaml trace↔method interop).

**Side-finding (JIT-unrelated, separate cheap win):** `byte_offset_to_position` is 7.7% inside the parse fold — parser likely rescans source from start per token for line/col (suspected O(N²) on big preludes); precompute a line-offset table once. See [[jit_aot_replay_section_3_5]], [[adsmt_emit_system]].

**Phase 3 design panel finding (CRUCIAL, 4 proposals × 3 judges):** a portable meta-method has **≈0 measured wall-clock payoff on the current production path** — the verdict path is the digest short-circuit (O(1)); the production slim trace is `[Restart, Conflict]` carrying NO prelude events to factor; the full-trace warm-start replay is currently DISCARDED (GuardsPassed→fresh solve); and the one measured lever (~6ms re-intern) is engine/global-hash-cons-bound (`TERM_CACHE` per-process Weak), so cross-process Arc adoption is impossible. So a meta-method here is NOT a speedup — its value is soundness-discipline + forward architecture. User chose "scoped sound architecture" (over minimal-footgun-only / redirect-to-prepare_cdcl_seed-caching / defer).
