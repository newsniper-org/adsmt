**Clean-room quantifier-instantiation engine at `~/oxiz-mbqi`** (separate local git repo, started 2026-06-12), to be ported into OxiZ replacing the patched-but-still-unsound `oxiz-solver/src/mbqi/`.

**Why:** OxiZ's MBQI produces spurious `unsat` on the real Verus `-V adsmt` prelude via MULTIPLE independent defects, and they can't be reliably delta-debugged because the bisection oracle (OxiZ's own subset verdict) is itself unsound (a superset reports `sat` while a subset reports `unsat`). Patching is whack-a-mole. The verus-fork P0 ("prelude false-unsat") A/B/C round was followed by a second round (`.local-replies-from/verus-fork/2026-06-12-P0-ABC-fixed-but-full-prelude-still-unsat.md`): A/B/C confirmed fixed in isolation BUT the FULL prelude still unsat via triggers D (trigger-free Height partial-order, distinct-bearing) and E (11-axiom distinct-free int-width/has_type) — and behind those, more (the fuel machinery). z3 times out goal-free on the full prelude, so the gate is "not unsat" (unknown OK).

**THE invariant that makes the rewrite tractable:** the engine NEVER concludes `Unsat`. It only emits sound, guarded, ground instantiation lemmas `Q ⇒ φ[x̄↦t̄]` where each tᵢ is a real ground term from a **ground-term index** (never a fabricated witness, never a bound variable), capture-free, guarded by the quantifier's boolean literal `Q` (so a `∀` under `(=> g …)` inherits the guard). `Unsat` is then ONLY the host ground SAT+theory core's → it's real → spurious unsat is structurally impossible. Every OxiZ defect reduces to "emitted an invalid lemma": B (self-match `{i↦i}`), E (sibling-capture `{i↦x_sibling}`), D (fabricated universe `u!N`), fuel (guard dropped). [[jit_aot_replay_section_3_5]]-style: the firewall is a single central check in `instantiate`.

**Architecture:** solver-agnostic `TermLang` trait → develop/z3-cross-check standalone (toy host), then port by impl'ing the trait for `oxiz_core::ast::TermManager`. Modules: term/ground/instantiate/engine/toy. Candidate-selection STRATEGIES (all feed the same sound `instantiate`, so all inherit the invariant): enumerate (M1 baseline), trigger/e-match (M2), **CDQI** (M2 — conflict-driven, picks ground tuples whose body is false under the model; re-derives OxiZ's `generate_ground_conflicts` inside the invariant), model_based (M3 — Ge–de Moura, witnesses used ONLY to evaluate, NEVER turned into lemmas = the D-bug firewall). Corpus: (1) soundness triggers (B/E/D/fuel/partial-order + full prelude), (2) **import OxiZ's z3-parity regression** (bench/regression + lia_/uf_/mbqi_/quant_ tests) so completeness is guarded. Oracle is z3, never OxiZ.

**GROUND AUDIT RESOLVED (2026-06-12) — three OxiZ-core soundness bugs fixed, fuzzer now 6000-problem / 4-seed / 100%-z3-agree / 0-unsound:**
- **EUF `use_list` OOB panic (robustness/soundness crash)** — interning a term DURING a theory check (after a SAT push) grows the EUF node space (`nodes`/`use_list`/`uf` via `intern`'s `uf.add()`), but EUF `pop` truncated `nodes`/`use_list` while `UnionFind::pop()` only undid unions, never shrank `parent`/`rank` → the NEXT `intern` desynced the node index from the uf slot and `find` returned a stale root past `use_list`. Fix (`oxiz-theories/src/euf`): drop path compression on the backtracked paths (`find_no_compress`/`same_no_compress` everywhere in `solver.rs`; union-by-rank alone keeps finds O(log n)) + new `UnionFind::truncate(num_nodes)` called in EUF `pop` to re-sync uf with nodes + bounds-safe `are_equal`/`find`/`*_immutable` (a popped/stale index = its own singleton, never panics).
- **Bug (b) injective deep-bounded spurious-UNSAT — ROOT CAUSE: the proof forest was not backtracked with the union trail.** EUF `pop` truncated `proof_forest` by node INDEX but left merge/congruence edges that had leaked onto SURVIVING nodes (< num_nodes) during the popped scope. `explain_equality`'s BFS then routed through a retracted-merge edge and returned reasons for equalities that no longer held → an INVALID (over-strong) learned conflict clause → spurious unsat. It only bites at scale + with backtracking (nested-app instances fire extra congruence → more leaked edges; 102 nested instances unsat, every small subset sat). Fix: `proof_trail`/`proof_trail_limits` (mirror `sig_trail`) record every proof-edge insertion via `push_proof_edge` and `pop` undoes them BEFORE `proof_forest.truncate`; also clear `expl_cache` on `pop` (it was only cleared in `merge`, comment lied). Found by **categorical bisection** (split the corpus by structural class → "≥1 arg is `(f n)`" = nested apps was the trigger) after realising **every subset of a SAT assertion set is SAT, so ddmin needs NO z3 oracle** (subset spurious-unsat ⟺ OxiZ says unsat).
- **Bug (a) nested-app spurious-SAT — RECLASSIFIED as SOUNDNESS and FIXED by REMOVING the guard.** `g(10)=5 ∧ f(g(10))=20 ∧ f(g(10))≤10` is UNSAT (20≤10 false) but OxiZ said Sat because the `has_conflicting_apply_arg` guard DROPPED the numeric nested app `f(g(10))` from arith (both `track_theory_vars` and `extract_linear_terms` in `encode.rs`), losing its direct bounds. The earlier conclusion that removing the guard "reintroduces spurious unsat (1-3/400), so keep it as a completeness-vs-soundness trade" was WRONG — that spurious unsat was a **symptom of bug (b)** (the proof-forest corruption broke the EUF↔arith congruence explanations). The congruence that keeps `f(g(a))` agreeing with `f(v)` once `g(a)=v` is supplied soundly by `propagate_euf_equalities_to_arith`, not by hiding the term.
